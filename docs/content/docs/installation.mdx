---
title: Installation
description: Install and configure Better Auth Ruby.
---

<Steps>

<Step>
### Install The Gem

For plain Rack or another Rack-based app:

```ruby title="Gemfile"
gem "better_auth"
```

For Rails:

```ruby title="Gemfile"
gem "better_auth-rails"
```

For Hanami 2.3+ apps:

```ruby title="Gemfile"
gem "better_auth-hanami"
```

For Roda apps:

```ruby title="Gemfile"
gem "better_auth-roda"
```

Then install dependencies:

```bash title="Terminal"
bundle install
```
</Step>

<Step>
### Set Environment Variables

Set a high-entropy secret and the public base URL of your app.

```txt title=".env"
BETTER_AUTH_SECRET=
BETTER_AUTH_URL=http://localhost:3000
```

You can generate a secret with:

```bash title="Terminal"
openssl rand -base64 32
```

<Callout type="warn">
Use a real secret outside test. The Ruby port allows the default test secret only
in test environments.
</Callout>
</Step>

<Step>
### Create A Better Auth Instance

```ruby title="config/auth.rb"
require "better_auth"

auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL", "http://localhost:3000"),
  email_and_password: {
    enabled: true
  },
  database: BetterAuth::Adapters::Memory.new
)
```

The memory adapter is useful for development and tests. Production apps should
use PostgreSQL, MySQL, the Rails ActiveRecord adapter, or the Hanami Sequel
adapter.
</Step>

<Step>
### Configure Persistence

PostgreSQL:

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: BetterAuth::Adapters::Postgres.new(
    url: ENV.fetch("DATABASE_URL")
  )
)
```

MySQL:

```ruby title="config/auth.rb"
auth = BetterAuth.auth(
  secret: ENV.fetch("BETTER_AUTH_SECRET"),
  base_url: ENV.fetch("BETTER_AUTH_URL"),
  database: BetterAuth::Adapters::MySQL.new(
    url: ENV.fetch("DATABASE_URL")
  )
)
```

Rails apps normally use the generated ActiveRecord adapter configuration:

```bash title="Terminal"
bin/rails generate better_auth:install
```

For Hanami:

```bash title="Terminal"
bundle exec rake better_auth:init
bin/hanami db migrate
```

For Roda:

```bash title="Terminal"
bundle exec rake better_auth:install
bundle exec rake better_auth:generate:migration
```
</Step>

<Step>
### Mount The Handler

Rack:

```ruby title="config.ru"
require_relative "config/auth"

map "/api/auth" do
  run auth
end
```

Rails:

```ruby title="config/routes.rb"
Rails.application.routes.draw do
  better_auth
end
```

Roda:

```ruby title="app.rb"
class App < Roda
  plugin :better_auth

  better_auth

  route do |r|
    r.better_auth
  end
end
```

The default base path is `/api/auth`.
</Step>

<Step>
### Verify The Server

Send a request to the health endpoint:

```bash title="Terminal"
curl http://localhost:3000/api/auth/ok
```

Expected response:

```json
{ "ok": true }
```
</Step>

</Steps>
